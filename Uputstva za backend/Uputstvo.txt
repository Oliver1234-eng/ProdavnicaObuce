Prvo treba instalirati, ko nema, JDK za Javu (ja imam verziju 21) i Eclipse IDE for Enterprise Java and Web Developers (ja imam verziju 2025-09). Kada se projekat povuce sa GitHub repozitorijuma pokrene se Eclipse (eclipse_workspace) pa ide se gore levo na File-->Import-->Maven-->Existing maven project, onda za Root Directory ide se na Browse i pronadje se Java projekat skinut sa Githuba, sa tim da poslednji deo lokacije bude "/backend" jer ce onda prepoznati i stiklirati pom.xml i onda Finish. Posle treba neko vreme na Maven prepozna i ucita projekat. Treba onda skinuti MySQL bazu, ko nema, (ja imam verziju 8.0.44) i u toku instalacije ako pita za lozinku MySQL servera treba napisati "root". To ce kasnije biti vazno. Kad se pokrene MySQL onda se ide na lokalnu instancu localhost:3306, pa ukucati lozinku "root" i otvorice se. Gore levo se ide na sliku za nov SQL Query i onda ukuca se "create database prodavnica_obuce". Tabele ne kreiramo to ce Spring Boot preko JPA anotacija sam da kreira. Sad se vracamo na nas projekat, u src/main/resources ima fajl application.properties koji ovako izgleda:

spring.application.name=prodavnicaobuce

#spring.jpa.hibernate.ddl-auto=update
spring.jpa.hibernate.ddl-auto=validate
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
logging.level.org.springframework.security=DEBUG

spring.datasource.url=jdbc:mysql://localhost:3306/prodavnica_obuce
spring.datasource.username=root
spring.datasource.password=root

Ova poslednja linija (spring.datasource.password=root) ostaje ovako ako ste prilikom instalacije MySQL-a za lozinku stavili "root", ako niste to se menja sa vasom lozinkom koju ste postavili. A najvaznije je sledece: PRE NEGO STO PRVI PUT POKRENETE PROJEKAT OVO TREBA DA JE OTKOMENTARISANO spring.jpa.hibernate.ddl-auto=update A ONO DRUGO ZAKOMENTARISANO spring.jpa.hibernate.ddl-auto=validate ZNACI OBRNUTO OD OVOG KAKO JE SADA. Sada pokrenuti projekat, sacekati u konzoli da pokrene projekat i kreira tabele i ako je sve u redu odmah prekinite izvrsavanje projekta. Vracate se na application.properties i zakomentarisete ovo spring.jpa.hibernate.ddl-auto=update a otkomentarisete ovo spring.jpa.hibernate.ddl-auto=validate (za komentarisanje stavi se ispred linije #) i posle svaki sledeci put kad pokrenete projekat ovo se ne dira vise. Ako slucajno hocete da obrisete bazu i opet kreirate u MySQL-u se kuca komanda "drop database prodavnica_obuce" pa "create database prodavnica_obuce" i onda ovaj postupak za application.properties se ponovi da bi opet kreiralo tabele.
